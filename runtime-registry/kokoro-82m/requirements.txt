# peakvox/kokoro-runtime requirements (R8 reference).

# Future runtimes (F5-TTS, XTTS, OpenVoice, Fish, OmniVoice)

# Pin strategy: exact version pins for reproducibility.

# lockstep with the kokoro framework version; the spec.image.digest
# is set by the build script after a successful `docker build`.

# /v1/generate, /v1/variants/build, /v1/metadata over HTTP/JSON.

uvicorn[standard]==0.32.0

# The Kokoro TTS framework.

# across 9 languages. The first runtime service PeakVox ships.

# Spacy is required by the kokoro package for phonemization.
# The en_core_web_sm model is auto-downloaded on first inference
# (or baked into the image during build for offline deployments).
